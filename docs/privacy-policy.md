# プライバシーポリシー (TimerUtility)

最終更新日: 2026-06-20
バージョン: 1.0 (Play Store 提出用確定版)
原本: 本ファイル (日本語)
英語版: [docs/privacy-policy.en.md](privacy-policy.en.md)

TimerUtility (以下「本アプリ」) は、ユーザーのプライバシーを尊重し、設計上の優先事項として
データ収集ゼロのアーキテクチャを採用しています。本ポリシーは、本アプリがどのような情報を
扱い、どのように扱わないかを明確にすることを目的としています。

---

## 1. 要約

- **本アプリはユーザーの個人情報を一切収集・送信しません。**
- すべてのデータ (タイマー、アラーム、世界時計の登録都市、各種設定、診断ログ) は
  ユーザーの端末内にのみ保存されます。
- ネットワーク通信機能は実装されていません (バックエンドサーバーは存在しません)。
- 広告 SDK、解析 SDK、クラッシュレポート SDK は同梱していません。
- 位置情報は世界時計機能の現在地タイムゾーン推定のためにのみ一時利用され、
  端末外には送信されず、永続化もされません。

---

## 2. 収集しないデータ (Data Safety 上の「収集なし」申告に対応)

本アプリは以下のいずれも **収集しません**:

- 個人を識別する情報 (氏名、メールアドレス、電話番号、住所、生年月日)
- アカウント識別子 (Google アカウント、SNS アカウント等)
- 端末固有 ID (IMEI、広告 ID、ANDROID_ID 等)
- 連絡先 / カレンダー / 写真 / マイク / カメラへのアクセス (権限自体を要求していません)
- アプリ利用統計 / クラッシュレポート / 解析データ
- 課金情報 (アプリ内課金を実装していません)

---

## 3. 端末内でのみ扱うデータ

以下のデータは本アプリの機能のために端末内のアプリ専有ディレクトリに保存されますが、
**端末外には一切送信されません**。

| データ | 用途 | 保存場所 | 削除方法 |
| --- | --- | --- | --- |
| タイマー設定 (本数、時間、ラベル、音源、スヌーズ設定) | 複数タイマー機能、起動時復元 | 端末内 SQLite (Drift) | アプリ内 UI、または Android 設定 → アプリ → ストレージを消去 |
| アラーム設定 (時刻、曜日繰り返し、有効/無効、音源) | 指定時刻アラーム機能 | 端末内 SQLite (Drift) | 同上 |
| プリセット | プリセット機能 | 端末内 SQLite (Drift) | 同上 |
| 世界時計の登録都市 (タイムゾーン識別子と表示順) | 世界時計機能 | 端末内 SQLite (Drift) | 同上 |
| ユーザー設定 (テーマ、言語、デフォルト音源、CVD モード、診断ログトグル等) | 設定画面の項目永続化 | 端末内 SharedPreferences | 同上 |
| 診断ログ (ユーザーが明示的に有効化した場合のみ) | テスター / 開発者向けの不具合解析補助 | 端末内アプリ専有ディレクトリ (JSON Lines、最大 14 日 / 50 MB / 1 ファイル 1 MB) | 設定画面のトグル OFF、またはストレージを消去 |

これらのデータはアプリのアンインストールで完全に削除されます (Android OS の標準動作)。

---

## 4. 位置情報の取扱い

本アプリは世界時計機能で「現在地のタイムゾーンを自動推定する」目的にのみ位置情報を
利用します。

- 権限: `ACCESS_COARSE_LOCATION` (粗い精度のみ、`ACCESS_FINE_LOCATION` は要求しません)
- 利用タイミング: 世界時計画面で「現在地を追加」操作をユーザーが明示的に行った時のみ
- 利用内容: 取得した緯度経度を端末内の `geocoding` API に渡して国コードを取得し、
  そこからタイムゾーン識別子 (例: `Asia/Tokyo`) に変換します
- **緯度経度の値は端末を離れません**。アプリ内のメモリ上で変換に使用された後は破棄され、
  Drift / SharedPreferences にも永続化されません
- 位置情報の利用をユーザーが拒否した場合は、端末のシステムタイムゾーン
  (`FlutterTimezone.getLocalTimezone`) にフォールバックします

---

## 5. 権限の利用根拠

本アプリが [AndroidManifest.xml](../android/app/src/main/AndroidManifest.xml) で
宣言している 8 つの権限はそれぞれ以下の目的でのみ使用します。

| 権限 | 利用目的 | 通知タイミング |
| --- | --- | --- |
| `ACCESS_COARSE_LOCATION` | 世界時計の現在地タイムゾーン推定 (§4 参照) | 「現在地を追加」操作時の OS ダイアログ |
| `POST_NOTIFICATIONS` | タイマー / アラームの通知表示 (Android 13+ 必須) | 初回タイマー/アラーム作成時の OS ダイアログ |
| `SCHEDULE_EXACT_ALARM` | 指定時刻アラームの正確な発火 (Doze 回避) | Android 14+ では設定誘導 |
| `USE_EXACT_ALARM` | Android 14+ で時計 / アラームカテゴリに認められる代替権限 | システム自動付与 (権限ダイアログなし) |
| `USE_FULL_SCREEN_INTENT` | ロック画面でのアラーム表示 (Android 14+ 事前申請要件) | Android 14+ では設定誘導 |
| `WAKE_LOCK` | アラーム鳴動時に CPU をスリープから起こす | 自動付与 |
| `VIBRATE` | バイブレーション (通知 / アラーム鳴動) | 自動付与 |
| `RECEIVE_BOOT_COMPLETED` | 端末再起動後のタイマー / アラーム自動復元 | 自動付与 |

---

## 6. 診断ログ (Diagnostic Logging) の取扱い

設定画面の「診断ログを記録」トグルを **ユーザーが明示的にオン** にした場合のみ、
本アプリは内部動作ログを端末内に保存します。

- 保存内容: タイマー / アラームの開始・停止・スヌーズ等のイベント、エラー情報、
  権限取得結果。**PII (個人識別情報) は事前にマスク** されています:
  - 位置情報の緯度経度は記録しません (タイムゾーン識別子のみ)
  - ユーザーが入力したタイマー / アラームのラベル文字列は記録しません
- 保存場所: 端末内アプリ専有ディレクトリ (JSON Lines、ローテーション: 14 日 /
  累計 50 MB / 1 ファイル 1 MB で分割)
- 共有: ユーザーが「診断ログを共有」ボタンを押した場合のみ、ログを zip にまとめて
  Android OS の Share Sheet で **ユーザーが選択した宛先** (メール / Google Drive /
  メッセンジャー等) に渡します。**自動送信は一切ありません**
- 削除: 設定画面のトグルを OFF にすると新規記録は停止 (既存ファイルは保持)。
  完全削除はストレージを消去 / アンインストールで行えます

## 7. 第三者サービス

本アプリは **第三者の解析 / 広告 / バックエンドサービスを一切利用していません**。

同梱しているサードパーティライブラリ (Flutter SDK、Riverpod、Drift、
flutter_local_notifications、audioplayers、permission_handler、geolocator、
geocoding、share_plus 等) は OS API を呼び出すクライアントライブラリであり、
外部サーバーへの通信は行いません (geocoding は Android 標準の `Geocoder` API を
利用し、Google API への通信を伴わない端末内辞書ベース)。

完全な依存リストは [THIRD_PARTY_NOTICES.md](../THIRD_PARTY_NOTICES.md) を参照。

## 8. 子どものプライバシー

本アプリは子ども (13 歳未満、米国 COPPA 基準) を対象に設計されていませんが、
**個人情報を一切収集しない** ため、子どもが利用した場合でもプライバシー上の
追加リスクはありません。

---

## 9. データの暗号化

本アプリは個人情報・認証情報・通信情報を扱わないため、暗号化の対象となるデータが
存在しません。Drift / SharedPreferences のデータは Android OS の標準的なアプリ専有
ディレクトリ暗号化 (端末ロック解除後にアクセス可能) に従います。

---

## 10. ユーザーの権利

本アプリは個人情報を収集していないため GDPR / CCPA / 個人情報保護法等のもとでの
「アクセス・訂正・削除」リクエストの対象となるデータが存在しません。

ユーザーが端末内のアプリデータを完全に削除したい場合は以下のいずれか:

- Android 設定 → アプリ → TimerUtility → ストレージとキャッシュ → ストレージを消去
- アプリをアンインストール
